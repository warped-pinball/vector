

Configs all have a 'Type' entry in the JSON.
Maybe should be alpha and be descriptive - but they are just numbers.
Here is what they mean- -


0: OFF, not implemented feature

1-8: Display Message - System 11
	Type 1
		Layout: 7 characters × 6 lines
		Encoding:
		Spaces and decimals: 0x20 (standard space)
		Letters/numbers: Normal ASCII (ord(char))
		Example: 'A' = 0x41, '0' = 0x30
	Type 2
		Layout: 16 characters × 3 lines
		Encoding:
		Spaces and decimals: 0x20 (standard space)
		Letters/numbers: Normal ASCII (ord(char))
		Same as Type 1, just different display dimensions
	Type 3
		Layout: 16 characters × 3 lines (split across 2 displays, 8 char each)
		Encoding:
		Spaces and decimals: 0x00 (null)
		Numbers: ASCII - 0x2F
			'0' (0x30) → 0x01
			'1' (0x31) → 0x02
			'9' (0x39) → 0x0A
		Letters: ASCII - 0x36
			'A' (0x41) → 0x0B
			'B' (0x42) → 0x0C
			'Z' (0x5A) → 0x24
		Decimal point: Add 0x55
	 
9: Display Message - System 9
	Type 9 (System 9)
		Purpose: Display IP addresses on System 9 pinball machines (which use numeric-only displays)
		Layout: 4 separate numeric displays (one for each octet of the IP address)
		Encoding: BCD (Binary Coded Decimal)
		IP address split into 4 numbers: e.g., "192.168.1.100" → [192, 168, 1, 100]

20-29: DataEast

	In-play scores
	20 four byte scores, 7 byte spacing
	24 has a break with fifth byte out of sequence at end
	25 five bcd coded bytes in a row
	27 seven digit score BCD one digit per byte
	
	
	High Scores
	20 all in a row, can have mulitplier of 10 optionally 
	24 five bytes has a break after fourth byte, offset fixed 16 bytes
	25 five bytes all in a row
	27 BCD one digit per byte in a row - seven digits
	
	Ball inplay
	type 20 - data east 0,1,2,3,4,5

	Display message
	Type 0: none/failure
    Type 20: 12x3 = 36 chars
    Type 21: 38x1 = 38 chars (RoboCop)
    Type 22: 14x3 = 42 chars  alpha 
    Type 23: 16x3 = 48 chars
	Type 24: 24x2 but only 21 in each line visible - put "Message21" in name.txt
